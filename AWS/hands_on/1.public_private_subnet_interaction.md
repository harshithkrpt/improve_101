## VPC Setup with Public + Private Subnet and Bastion Access

### 1. Create VPC

* Go to **AWS Console → VPC → Create VPC**
* Name: `demo-vpc`
* IPv4 CIDR block: `10.100.0.0/16`
* Tenancy: Default

---

### 2. Create Subnets

**Public Subnet**

* Name: `public-subnet`
* CIDR: `10.100.0.0/24`
* Enable Auto-assign Public IP = YES

**Private Subnet**

* Name: `private-subnet`
* CIDR: `10.100.1.0/24`
* Auto-assign Public IP = NO

---

### 3. Create Internet Gateway

* Name: `demo-igw`
* Attach it to `demo-vpc`

---

### 4. Route Tables

**Public Route Table**

* Create route table: `public-rt`
* Associate → `public-subnet`
* Add Route:

  * Destination: `0.0.0.0/0`
  * Target: Internet Gateway (`demo-igw`)

**Private Route Table**

* Create route table: `private-rt`
* Associate → `private-subnet`
* Default route remains:

  ```
  10.100.0.0/16 → local
  ```

(No internet route added intentionally)

---

### 5. Launch EC2 Instances

**Public Instance (Bastion Host)**

* Subnet: `public-subnet`
* Auto Public IP: Enabled
* Security Group:

  * Allow SSH (22) from your IP
  * Allow ICMP (optional for ping)

**Private Instance**

* Subnet: `private-subnet`
* No public IP
* Security Group:

  * Allow SSH from **public instance private IP** or **public subnet CIDR**

---

### 6. Connect to Private Instance via Public Instance

Step 1 — Connect to Public EC2

```bash
ssh -i key.pem ec2-user@<public-ip>
```

Step 2 — From Public → Private

```bash
ssh -i key.pem ec2-user@<private-ip>
```

---

### 7. Architecture Logic

Public instance acts as a **jump host (bastion)**:

* Internet → Public EC2 → Private EC2
* Private instance is **not exposed to internet**
* Only reachable internally

This pattern is used in production for:

* security isolation
* database tiers
* internal services

---

### 8. Validation Checklist

✔ Public EC2 accessible from internet
✔ Private EC2 has no public IP
✔ Private EC2 reachable from Public EC2
✔ Private route table has no IGW route
